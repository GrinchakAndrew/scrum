Section-1: Scope Management

  • Control and monitor the scope of Delivery of Projects through the medium of:
    • -- Fixed and Communicated Deadline for Base-Scope of the Project
    • -- Setting Due dates on Tasks of the Projects
    • -- Risk assessment and estimation of overheads
    • -- Establishing deadlines and red-lines
    • -- Coordinate final deadline approval and Risks Estimations
    • -- Participation in iterative deadlines confirmation communications
    • -- Scheduling each sprint and setting up calendar reminders
    • -- Re-scheduling sprints and informing all stakeholders about such initiative
  • Confirming established deadlines on all the project with all the involved stakeholders.
  • Keeping track-records and references to the additional tasks for the purpose of reporting them on request.
  • Coordinate created bugs to their due dates, setting due dates on bugs according to development camp workload.

Section # 1 Details:

Scope management means working with the schedule of the project as it related to tasks and project deliverable increment that should be a result of each sprint. Scope should be divided into equal sprints and planned according to breakdown of tasks into workflows of the project. To do this, Scrum Master should communicate with the Technical Analysts and work together with the team to define the ETA of each task against the general timeline of the project.

Scope:

  • - Tasks breakdown into Dev Flows
  • - Tasks ETA
  • - Sprints breakdown
  • - Deadline ETA
  • - Redline ETA
  • - Sprints ETA

List of Items to Create\Update:

  • - Project Scope \ Timeline
  • - Tasks ETA
  • - Sprints breakdown
  • - Letter of notification to stakeholders about documented Scope
  • - Project wireframe created in task tracking system (e.g. Asana)
  • - Tasks added to the project in Asana
  • - Stakeholders added to the project in Asana
  • - All needed docs are shared to stakeholders

Total documents-to-create\update: 3

Section # 2 : Reporting

  • - Making status reports on the state of the project at any given stage of the project.
    • - Keeping track of bugs for any given project for the purpose of reporting.
    • - Planning and scheduling communication events with the Client on any current project issues with the purpose of making reports

Details:

Reporting on the status of the project implies tracking tasks dynamic progress, and providing percentage of completion rate of tasks.

Report:

  • - Percentage of completed tasks vs. incomplete
  • - Pace of project completion
  • - Expected timing of the next build
  • - Amount of open bugs
  • - Amount of unanswered questions about tasks

    • List of Documents to Create\Update:

    • - Bug list
    • - List of completed tasks
    • - List of incomplete tasks
    • - Percentage of project completion
    • - Letter of report
    • - Questions about the complications or unclear requirements

    Total documents-to-create\update: 5
